工作半年了! 紀錄一下心得。


人生中的第一次 KPI 約談

2020/8/11 經歷人生中的第一次 KPI 約談，與經理一對一談話
待的部門是開發部，專案有分開發專案和系統專案
開發專案只是跟其他部門推廣用的(幫別部門翻新舊系統變成 asp.net core 和 angular 架構)
系統專案就是真的有 PM、SA、QA、RD 等一起開發新系統
現在只寫過開發專案的兩個前端功能畫面
經理跟我說等三四個專案完就會把人轉出去別的部門
我問有什麼準備方向 他說除了要對基礎框架熟悉以外
還要知道如何把做好的系統放到 VM 上 ，而 docker 現在只有一個部門在用

談完後跟同事說我覺得超可怕的QQ
同事很有遠見的跟我說不用去記每一行程式碼是怎麼寫的
要去記整個架構、流程，知道什麼東西會去哪，不要太執著在技術
因為其他部門不一定是用我們科的技術，
主要是要知道想要這個功能要往哪裡找，你就會寫了

真心覺得有同事可以問問題真是太幸運了QQ

覺得我一點也不在乎 KPI
比較在乎的是能不能在做專案和轉部門時繼續活著

然後真的覺得可以報名到程式導師計畫真是太幸運了
至少我會有些常識可以應戰 也覺得要是失敗了也不怕找不到工作xD

參加導師計畫與參與專案

截至今日我交了 w1-w4、w6 作業，從 w1 的 CLI、Git、w2/w3 的 OJ、w4 的網路概論與接 API、w6 的 SEO、CSS 面試題、切版、w7 的 JS 事件
覺得有入門的機會蠻好的 我一定要把握好這些稍縱即逝的機會!!
專案的話接觸到 RWD 感覺很棒，要把 SASS 的 @media query 整理起來才行!!
想辦法讓自己下一次速度更快!!!

所以接下來的努力方向

最近有點彈性疲乏
仔細想想主因 應該是看太多理論和做太多作業了xD
接下來為了找回初心想把以前的東西挖回來再做一遍
學習上則是多跟生活上的例子做連結、才能加深印象記的久，
不然都是理論和作業實在太無聊拉rrrrr 是不是?xD
番茄鐘、輪撥牆，或是做一下之前想做的 mp3 播放機，gogogo~~
