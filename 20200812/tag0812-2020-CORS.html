Preflight Request 是什麼?

Preflight Request 原來是為了…
request 完回來的 response，雖然因 CORS 被瀏覽器攔截，
但其實這個請求已經被接收、已經對 Server 造成影響(也許對 db 改資料了)
所以才會先來一個 Preflight Request 確認是否有 CORS(cross origin resource sharing) 問題，
有的話就不會發接下來的 request，這樣就不會造成已經到 Server、東西回來還被瀏覽器擋不見…
沒有的話就照常發 request
可是這樣每次都要先檢查好像很神經質
所以把請求分成簡單請求和預檢請求
檢查會發生 CORS 的請求，而不是通通都檢查
(不是像韓國那樣全國普篩，而是像台灣現在菲律賓歸國的特別篩…)
