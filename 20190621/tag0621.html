練習用 Vue

一直都只是用 jquery 操作 DOM、用重新渲染 DOM 覆蓋 DOM 元素的方式更新資料。
用完這個+1-1好像有點懂”操作資料”的 Vue 了!


See the Pen
vue的.self/jq$(this) by kiwijang (@kiwijang)
on CodePen.

覺得可能是因為不太了解自己定義 html 裡的屬性和值，所以不太知道 v-for跟javascript的關西，
譬如 <button> v-for="(item,key,index) in items" @click="getArg(item,key,index)" </button>
當 items 是陣列，可用(物件,index) in items 引用參數(物件,index)到 js 裡 Vue 使用 / (物件,index) 直接在 html 裡用，
當 items 是物件，可用(value,key,index) in items 引用參數(value,key,index)如上。

概念是把寫在 js 的資料 → 引到 html 用 → 利用 v-bind、v-model、v-bind…將資料與 DOM 元素綁起來，
v-for、v-if…在 html 做迴圈與判斷 → 在 html 產生變數，藉由設定在 methods 與 computed 的函式，將在 html 裡產生的變數以參數傳入函式使用 → 在 js 產生的值會要改變 Vue({data: {//…}//…}) ，也就是 Vue物件裡的 data

所以 html 就會根據改變的 data 改變資料了~^^

在這篇文章 在vuejs中是否还需要jquery 裡說到:

Vue 的官方是不建议直接操作 DOM 的，Vue 的用途在于视图和数据的绑定。如果通过 JQuery 直接操作 DOM 的话，势必会造成视图数据和模型数据的不匹配，这样 Vue 就失去它存在的意义了。

JQuery 和 VueJS 合理使用并不会造成冲突，因为他们的侧重点不同，VueJS 侧重数据绑定和视图组件，JQuery 侧重异步请求和动画效果。

如果使用JQuery + VueJS 开发，一定要在 Vue 渲染完所有的 HTML组件之后再通过 JQuery 处理，而使用 JQuery 时应避免直接操作 DOM，但是应用动画是允许的。

JQuery 与 VueJS 相互配合可以非常高效的完成异步任务，首先通过 JQuery 发出 Ajax 请求，接受到服务端传递的 JSON 数据后，再通过 Vue 将数据绑定到组件上，最后由 JQuery 进行动画处理，整个过程就如行云流水般自然。

说句题外话，Vue 的目的不是取代 JQuery，它是为了解决前后端分离而出现的。如果没有数据变化，只是单纯的样式变化，则没有必要去大费周章进行视图模型的绑定，并且还不利于 SEO 优化。
