發表於 2019-05-09 | 更新於 2020-05-17 | 分類於 學習日記

日記

把ajax快速閱覽過一次了，後面提到 ajax順序

  • ajax順序

    1. 創建請求對象
      (從瀏覽器API new一個XMLHttpRequest())
      書中還有做不同瀏覽器的請求判斷.. 如果是微軟舊的瀏覽器要 new 這個..(難怪有人這麼討厭xD)
      new ActiveXObject(“Msxml2.XMLHTTP”)
      new ActiveXObject(“Microsoft.XMLHTTP”)
      →
    2. ☞ request = new XMLHttpRequest() <- 存成全域變數，回調函數才可使用
      請求對象的準備狀態，
      存在 readyState 屬性中
      readyState: 0
      連結尚未初始化
      →
    3. 回調函數
      ☞ request.open(“GET”,url,true)
      ————-如何連接?連去哪?同步(true)/異步(false)
      readyState: 1
      連結初始化:請求已準備好可以發送
      →
    4. ☞ request.send(null)
      (給伺服器)—–這邊設定伺服器回傳null
      readyState: 2
      (伺服器正在處理請求)
      →
    5. readyState: 3
      (得到伺服器響應)
      數據下載到請求對象(瀏覽器)，但響應還沒完全準備好，還不能使用
      →
    6. readyState: 4
      (伺服器響應準備就緒)
      伺服器處理完請求，數據可以使用了

    6.☞ 利用request.onreadystatechange = function executeMe(){}每次readyState改變時就會執行這個指定的函數。
    可用 readyState 1.2.3.4 來在不同階段執行想設定執行的函數
    ex. 以 request.readyState ==4、request.status ===200 判斷式，來對已接收完成的資料作運用，然後
    ☞ request.responseText 取得回傳的文本

    • .responseText 對沒接觸伺服器端過的我來說很抽象，待補坑。

    XMLHttpRequest物件(對象)

    的屬性有:readyState、responseXML、responsText、onreadystatechange、statusText、status
    服務器響應會存在 responseText 中。通常是文本，不過也可以是XML數據。

  • 事件處理程序(用addEventListener()DOM2中可為單個事件指定多個事件處理程序)

    • 異步應用

      有夠難，說了一個表單要異步2次，中間埋了一個監聽器函數、來確認異步伺服器回傳後是否都通過標準、最後submit送出鈕也發了一個同步請求，意味著資料送達伺服器前你都要等待完才能做別的事，這時怕太無聊他在同步請求前運行一個setInterval(撥放圖片,50)，雖然不能做別的事，但是畫面50ms會撥放一張圖)

  • DOM操作
  • XML
  • CSV
  • JSON
    • eavl()、JSON.parse()，不熟
  • 表單驗證(表達式應用)
  • POST
    • 對瀏覽器端得請求解碼要使用:
      setRequestHeader() 伺服器用響應首部向瀏覽器發送訊息。
      setRequestHeader(“Content-Type”,”applocation/x-www-form-urlencoded”) 告訴伺服器收到名/值，類似於 Web 表單提交時發送的名/值。
  • 開一 utils.js 放常用到的小工具函式(藉此增加自己常用的函式庫)
