為最近雜亂的 C# 基礎做一個整理…





第一個月


C# 培訓 - 先看書、後練習、最後考試

書本章節與學習概要:
型別、變數、常數、列舉與運算子
了解實質型別與參考型別的差異。

- 流程控制
練習使用 while、for 迴圈與 if 判斷。

- 陣列
了解一維陣列、多維陣列、不規則陣列(陣列的陣列)。

- 方法與屬性
了解存取修飾詞、參數傳遞方式與使用屬性存取質。

- 例外處理
使用 try catch 捕捉例外。

- 類別、物件與結構
了解 class 是物件的藍圖。abstract class 是無法實體化的類別 (即無法透過 new 關鍵字產生實體)。
物件就是new 關鍵字產生的實體。

- 繼承、介面與多型
使用 override 與 new、virtual關鍵字。了解介面的多重繼承。

- 泛型與 Iterator
知道泛型與 Iterator 的應用實作。

第一次線上測驗錯誤題目主要是:
編譯/執行發生錯誤
==== (寫程式碼時的紅綠蚯蚓) ====
而最開始C# 編譯器進行編譯的過程我們稱為「編譯時期」。
====== (按下執行按鈕) =======
而到CLR執行我們的程式(包含JIT編譯)我們稱呼為「執行時期」。

- 轉型
同型別可放到同型別。
每個型別儲存的大小(位元組)有限，小的可以放到大的，大的放到小的會有溢位、精準度問題。

- 捕捉發生溢位錯誤(checked)
若有溢位會拋出 OverflowException 例外。

- 陣列

第一次線上測驗錯誤題目主要是:
陣列宣告
int[] N = new int[3];
int[] N = {1,2,3};
int[] N = new int[]{1,2,3};
int[] N = new []{1,2,3};
int[] N = new int[3]{1,2,3};

- namespace、存取修飾子、類別
namespace: 用來控制類別和方法名稱的範圍。
存取修飾子: 用來控制命名空間、類別、成員可以在哪裡被取用。
類別: 類別可以是靜態的。

- 參數傳遞
在呼叫參數有 ref 和 out 的方法時，
● ref 參數要先初始化(out 不用)。
● out 一定要修改傳入的參數(ref 不用)。
● ref 需要在執行前初始化參數(給值)而 out 是在程式結束前需要初始化參數(給值)。

## 主要問題

命名規則:
欄位、變數開頭小寫。
參考科內的程式碼撰寫規則。
練習用迴圈排序。
要給初始值。
轉型很容易有溢位問題。應該要根據該型別大小來限制使用者輸入的數字大小，或給予空間更大的型別。如: double 換成 decimal。
要包 try catch 以免程式直接停止。
練習方式與流程
每日使用TFS更新自己工作面版。
Effort有三種:
1. 原始預估: 寫自己預估多久完成。
2. 剩下: 寫自己剩下多久時間可完成。
3. 完成: 完成時總共花了多少時間。
依據題意使用c# 解題。
透過 SharePoint將完成的檔案上傳到新人上傳專區。並利用 Jabber通知指導員已完成。
透過TFS管理 Bug修改: 指導員在工作面板設置 Bug 面板，並親自到座位旁一一說明錯誤。
更新後再次上傳檔案至SharePoint。
循環3. ~ 5. 直到沒有錯誤為止。
小結:
自己常常以為檢查後沒有錯誤了，但繳交後卻還是有很多錯誤。希望自己可以再細心一點，題意若有疑問或感覺不對勁，應該先去跟指導員做確認，不應以自己認知撰寫。

- 問題四十四：按各科平均成績從低到高和及格率(85分以上及格)的百分數，從高到低排序。

100count(case when DEGREE >= 85 then 1 else null end)/count() as ‘及格率’

- 問題十八：查詢exam_grade表中，所有同學的SNO、CNO和rank列，並按照rank列排序。
select SNO,CNO,
(case when [DEGREE] between 90 and 100 then ‘A’
when [DEGREE] between 80 and 89 then ‘B’
when [DEGREE] between 70 and 79 then ‘C’
when [DEGREE] between 60 and 69 then ‘D’
when [DEGREE] between 0 and 59 then ‘E’
end) as ‘rank’
from [dbo].[exam_score]
order by rank

## 學習重點
【JS】
expr1 || expr2
（邏輯 OR）如果 expr1 可以轉換為
true，就返回 expr1。否則返回 expr2。
【CSS】
Child和Descendant的差別
● Child 有上下層順序(譬如: div>p)
只會選到div的下一層p。
● Descendant沒有上下層順序
(譬如: div p)。會選到div裡所有
的p(不管這個p有沒有上一層)。
● Adjacent (譬如: span + p)
在同一層的意思。
