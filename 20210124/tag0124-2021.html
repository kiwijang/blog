sharepoint 2016 接手系統紀錄

發表於 2021-01-24 | 分類於 工作紀錄

覺得要把去年開始遇到的狗屎事記錄在這裡…(負能量注意)

前言

sharepoint 2016 是去年調到金控的 Oxx 留下的毒瘤…
我怎麼會說這是毒瘤呢?請讓我娓娓道來…
我到職後第 4 個月，E 前輩要調去別的部門，
於是坐在經理旁邊的菜鳥(我)，就「順便」接手了 E 前輩的東西。
簡單描述一下(我看到的)E 前輩之於此系統的關西，

  1. 接手 Oxx 的系統
  2. 去恆毅上過 sharepoint 的課
  3. 手動刪除 log 檔案
  4. 非常討厭 P 經理，說經理什麼都不知道

在我接手此系統時遇到的問題有:
log 造成磁碟太滿…要手動上去 server 清…
經理希望此系統下架…
然後我對此系統什麼都不知道…
(不知道: server 有哪些?使用者有哪些?功能的使用流量?這個系統的用途是什麼?)

然後照我們公司冗長的行事作風:
花了一個禮拜寫電聯單申請環境(AP、UAT)的 VDI，
花了一個禮拜跟我交接東西，
E 前輩帶我用我電腦跟我說明:
先跟 NTA 用 jabber 溝通幫忙登入 AP 環境(先用 VMware 登入可登入 AP 的主機，再遠端連到 AP 上)
然後進去某個資料夾清裡面的 log 檔案………
(乾 那時我連 NTA 是啥都不知道，就網管拉
幹，難怪聽聞隔壁科下去 6 個不久後都離職，
在這個年代，連一個能跟下屬說明公司部門的分工職責的經理都沒有，
反正我們公司很有錢，有錢到什麼都沒有………….)

馬的 那位 Oxx 真的很沒品，創建了一個會記 Log 但不會清 Log 的系統。
而且這個系統還他媽的沒有管理辦法…………..

那個時候樓下某科要辦活動需要頁面………..
直接用 Sharepoint 創建了一個頁面…………….

然後 log 開始每二個月要清一次……
接下來因為沒辦法管這個系統，也不希望再投入更多資金去維護這個系統
就希望下架這個系統…………

然後我覺得 E 前輩最靠杯的地方來了，因為她有上過 sharepoint 的課，
理應知道 sharepoint 的權限控管要如何拔掉……..
但她從不覺得要聽經理的話把那些權限拔掉……
然後就一直裝傻到她調去樓下。

我當然要拔權限讓這個失控的系統下架
結果拔拔拔拔到會議室租借功能 xd
然後大家都打去她那裏反應問題
她就憑記憶把權限加回去…………….
然後寄了一封信給經理和打電話給她的人說是我用壞的 xD
事實確實是如此(我把頂層權限拔掉了，會議室功能依附在頂層權限上面)，
但是一個大公司可以讓一個到職 4 個月(非本科菜鳥)的人癱瘓會議室租借功能絕對可以上新聞了吧。
→ P 經理繼續裝傻

反正那時覺得我他媽就跟他拚了，最糟就是拿資遣單。

我的難點

然後我們的 P 經理在我寄信給還在使用系統的人後，開始幫忙打電話給各個部門(經理永不跑第一，也不說去做、不要去做等需要負責的事)…..
簡述一下(我看到的)P 經理立場:
年資 22 年，一畢業就在這裡工作，是第一份工作也應該是最後一份工作。
一開始用 Fortran (打紙卡寫程式)，一路到了今天這個職位，他只想保住他的位子和面子。

覺得很佩服經理的脾氣，看他被其他科經理和協理當狗數落就覺得他很可憐…….
而且我一直覺得經理很細心、講話也很幽默…..(但心機很重)
照理來說我們科的技術長應該要挺他，可是沒有，常常會聽到技術長在背後數落經理…….
我覺得我們科是沒救了拉 哈
高層不支持高層
低層不體諒高層

那整個氛圍對我來說呢?
技術長在新人培訓的時候對我很好，總會關心我寫到哪裡寫的如何、給關鍵字….
我覺得會想變得跟他一樣厲害
經理會想替我著想(僅限於你對他是「有用的」狀態，基本上他就是一個沒 PM 功能、沒長官功能的人)
帶我的前輩很嚴格也很有原則(但被公司扭曲到心態很糟)

人們看起來都很好
但是為什麼兜起來會亂七八糟的?(主因應該是因為大家都沒事做，閒到每個人都在鬥薪水)…….

我們的 line 群組有兩個，一個有經理一個沒經理…..
我們前輩叫我不用去自學東西，公司都會讓你學…..

我…
當初好希望公司有讀書會、大家熱愛討論新技術，每天都可以寫程式碼。
但是公司最需要的是會跟人吵架的人、在冗長的流程裡不失去自我的人、懂得法律保護自己的人…….
經理因為沒有跟時勢充實自己，變成遇到事就把問題推給別人、從不自己做決定的雞掰人，
而且他還會說他以前過得有多好……..
可憐之人必有可惡之處。

看起來前輩如果繼續被動等公司給東西，以後也會變得跟經理一樣，
但是他們最喜歡抱怨經理。

我覺得他們很矛盾，明明行為跟經理一樣，又一直罵經理。
一直抱怨薪水低，但又不敢投履歷出去面試。

雖然我很討厭經理，討厭抱經理狗腿的人。
(經理從不關心我們專案時程，照理來說經理就如同專案經理 PM，要負責時程掌控、檢討、預算控管)
舉例來說，跟 K 組長和 J 組員一起做一個專案，我是中間加入的，
J 組員有一個月時間可以看 xUnit 和 nSubstitute 是啥，但她沒有….
要開始寫測試了，她連環境都架不起來，
我在水深火熱她還一直來拿我滑鼠，然後連 git push 都不會就算了，但她完全不想學 QQ
經過這次事件我想表達:我真的不比資管(沒熱誠)的人還差….

但是那次 PMD 約談 P 經理跟我說:你覺得 J 怎樣?M 怎樣?
完全沒問我怎樣…..當時就覺得這個團隊是不是有病，完全沒人在意做好的系統，
沒有事後檢討、沒有事前詳細的系統使用者調查、沒有時程安排。
讓我覺得非常難過，根本沒人在想到最前線用這系統的行員他們的使用狀況如何….

那陣子，J 問我薪水多少，T 也問我薪水多少
剛來公司不久 B 也問我薪水多少

你們他媽只在乎薪水的話怎麼還留在這裡???????????????????????

馬的超噁心，還有人直接嘴經理不懂技術。

是不是有病???????????????????????????????????????
(連國小生都會寫程式，你他媽在跩三小?????)

幹……………..

好離題ㄌ，
繼續回來講 Sharepoint
現在已經把系統使用者精簡到只剩下釘子戶 xdddd
等金控那邊的會議室和雲端建置好….
就可以下架這個系統了~

所以交接後我做了什麼?

  1. 將手動刪 log 改成執行批次檔刪 log。
    本想排程刪 log 但是申請作業流程太麻煩而作罷。(你要改個東西，要跟人報告還要求別人幫你改，他不會你還要指導他，真不懂為啥不讓我自己來就好????)
  2. 整理出 server 位置(因為 online office server 壞掉，整個系統架構還是一團謎… 因為 sharepoint 是網管幫忙搭建的….. 但出問題網管會說這是你負責ㄉ系統只有你知道~)
  3. 去看儲存容量設定，然後寄信跟經理講不用擔心。(因為他一直來問，我沒上過課但是還是有去翻找答案…..我覺得 E 前輩應該有做些設定，但她都不講，就讓經理空擔心)
  4. 接下來要修補漏洞(TLS1.0 和 SMB 希望可以成功不出錯 QQ)
  5. 嘗試去看使用者流量，發現最多只有一個月內哪個行編的人做了幾次請求的統計資料。(你還要自己去翻職員簿是不是有病?==可是有比沒有好啦….)

經理做了什麼?

經理只會出張嘴，表示他很擔心，以免出事可以說他之前就有提醒，不是他的問題~

本想年後離職，但是跟爸爸聊完覺得還是 8 月再離比較妥當 xdd
總之先開履歷。
而且我爸說: 為什麼不是他們先滾，而是你先走?
仔細想想也是。

保持距離，笑臉迎人，增強自己實力。讓他們氣死。

小結

  1. sharepoint 的系統架構當初建置時應該要有紀錄(但沒有)
    好險在 aws 上有提供架構圖可以參考…
  2. 我們公司部門將開發與網管分開
    但是管理 sharepoint 需要開發與網管同時進行….
    也就是說要有網管的權限才能用正常的 GUI 設定這個系統…
    但身為開發者，很多權限都沒有，導致很多東西都無法設定…
  3. 管理權限不易，因為只要那個子網域的管理者沒讓你進去看你的網頁，
    你就什麼都看不到(檔案、頁面、在裡面如何使用該網域)，
    所以如果要制定管理辦法，要請該子網域使用者定期回報使用權限狀態、使用容量。
  4. 讓接手的人(我完全沒經驗)心理汙染非常嚴重: E 前輩很愛抱怨 P 經理，然後說她自己有多厲害: Oxx 只跟他說 sharepoint 放在哪…其他都她自己摸出來的，
    最可怕的是他還會跟其他同事抱怨我如何處理，不過時間久了就知道了，我是不齒跟這種不敢直接跟我說的人為伍拉。
    (雖然有去上過課不是 E 前輩應該會要管理此系統的理由，但是 E 前輩交接東西的口氣真的還蠻靠杯爛的，更重要的是，我不相信全部是她自己摸出來的拉，哈哈)
  5. 為什麼大家都靜默? 因為這樣會得罪人，他們以後還要在這裡慢慢往上爬…………..

因為要查 sharepoint 東西意外發現向雲暉老師的溪洲資訊誌，應該是經過此事件之後的一絲暖流 QQ
我希望? 希望可以變強，然後以後要是有要做系統，管理辦法和架構圖一定都要有紀錄，不然後面接的人很可憐，超可憐，發生錯誤卻不知道要從哪開始是非常絕望的…
最後我覺得我們公司拆成網管和開發完全不適用 sharepoint 拉，幹，浪費我時間。
然後呢? 如果有機會一定要離開這個鳥地方~(不在乎使用者的爛地方，連自己的行員都不想顧好還顧得好顧客我是不相信拉)
那資訊部門的人的需求呢? 馬的 資訊人用 devops 好嗎? 規格書也能版控阿 QQ
然後我覺得上層的人腦袋一定有問題啦，自己招進來的人能力不行就算了還招到沒熱誠的….(因為他們自己就是沒熱誠的)
最好笑的是 P 經理還跟我抱怨誰誰誰不 ok，那不是你自己招進來的人嗎?讓他跟上團隊是你的責任吧?
你這樣見不得別人好還抱怨別人不好，那你到底要怎樣?(你只要面子跟位子哈哈哈 xDD)

在背後說人閒話我是不會做的，專注在眼前的事業吧，加油^^
