什麼是Http協議?

協議是指計算機通信網絡中兩台計算機之間進行通信所必須共同遵守的規定或規則。 ( HTTP抓包實戰 p1 )
HTTP 有很多應用，最著名的是用於 Web 瀏覽器和 Web 伺服器之間的雙工通信。 ( HTTP權威指南 )
目想了解的 HTTP 協議是 HTTP/1.1 版本。(HTTP/2 標準於2015年5月以 RFC 7540 正式發表，所以在 2015 以前的瀏覽器版本通常不支援)

如何以 Javascript 發送 HTTP 請求?

以 JavaScript 操作 HTTP，與後端 API 進行溝通。
最後，解析回應的 JSON 資料，並以之渲染頁面！
XMLHttpRequest — JavaScript 發送 HTTP 請求 (I)
jQuery Ajax — JavaScript 發送 HTTP 請求 (II)
Fetch API — JavaScript 發送 HTTP 請求 (III)

伺服器(sever)分:

以下截自 藍眼科技-何謂伺服器(Server) ?

  1. 伺服器(電腦)
  • 伺服器
    • 伺服器作業系統指執行在伺服器硬體上的作業系統。伺服器os 需要管理和充分利用伺服器硬體的計算能力並提供給伺服器硬體上的軟體使用。
  • PC機
    • 指個人電腦。
  1. 伺服器軟體
  • 伺服器軟體工作在客戶端-伺服器或瀏覽器-伺服器的方式，有很多形式的伺服器，常用的包括：
    • 檔案伺服器（File Server） - 如Novell的NetWare
    • 資料庫伺服器（Database Server） - 如Oracle資料庫伺服器，MySQL，PostgreSQL，Microsoft SQL Server等
    • 郵件伺服器（Mail Server） - Sendmail，Postfix，Qmail，Microsoft Exchange，Lotus Domino等
    • 網頁伺服器（Web Server） - 如Apache，thttpd，微軟的IIS等
    • FTP伺服器（FTP Server） - Pureftpd，Proftpd，WU-ftpd，Serv-U等
    • 域名伺服器（DNS Server） - 如Bind9等
    • 應用程式伺服器（AP Server） - 如Bea公司的WebLogic，JBoss，Sun的GlassFish
    • 代理伺服器（Proxy Server） - 如Squid cache
    • 電腦名稱轉換伺服器 - 如微軟的WINS伺服器

所以伺服器(硬體)->伺服器OS管理硬體資源並提供指令->伺服器軟體(由指令構成軟體，進行各種用途)

所以目前我是要了解從 client 端要發送請求給 server 端，先都全部以 server 端理解(不細分成伺服器硬體、OS、軟體)。

HTTP 旨在:

透過向客戶端(Client)提供與 資源種類 無關的統一介面(uniform interface)，藉此隱藏服務實現的細節。

同理:

伺服端 (Server) 不需知道每個客戶端的目的 (purpose)，單獨的考慮每個HTTP請求，而非僅與特定種類的 Client 或應用相關。Client 依賴的是抽象的介面 (HTTP) 與表示 (representation)，而非具體的 Server 或 特定的類型 (e.g., html、txt、json)，因此，不用也不該知道 Server 如何實作資源。
